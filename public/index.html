Cars Routes

Method URL Body Description
GET /Cars Get a list of all the Car, will present 5 results per page
GET
  1. /Cars?page=2
  2. /Cars?perPage=4
    3. Note: You can use the above for all GET methods
     Note: You can also combined the above for multi filter option
     (/Cars/?perPage=3&page=2)
  1. Get a list of all the cars, will present 5 results in page 2
  2. Get a list of all the car, will present 4 results in page
GET /Cars/search?s={keyword(Name,Info)}
(/cars/search?s=BMW)		 search for a car or a car description/info, will present 10 results per page
GET /Cars/category/{category name}
(/Cars/category/suv)		 Get a list of all the Cars in selected category, will present 4 results per page
GET /Cars/price?min={value}&max={value}
(/cars/prices/?min=160000&max=1000000)		 Get a list of all the cars priced between 160000 and 1000000, will present 10 results per page (you can add only minimum price or maximum as well
/cars/prices/?max=1000000)
POST /Cars 
{
 "name": "Car name",
 "info": "Info about the car",
 "category": "Category name",
 "img_url": "image.jpg"
 "price": price,
}
Add a new car to the data base, You need a token for this query
PUT /Cars/{_id} 
{
 "name": "Car name",
 "info": "Info about the car",
 "category": "Category name",
 "img_url": "image.jpg"
 "price": price,
}
Update a car in the data base,You need a token for this query (only admin users alowed to modified other users entry)
DELETE /Cars/{_id} Delete a car from the data base, You need a token for this query (only admin users alowed to delete other users entry)

Users Routes

Method URL Body Description
GET /users Get a list of all the users, will present 10 results per page, You need a token for this query (only for admins)
GET /users/userInfo Get a user info, You need a token for this query (only for current user)
POST /users 
{
"name": "Name",
"email": "Email addres",
"password": "example"
}
Add a new user to the data base
POST(login) /users/login 
{
"email": "example@example.com",
"password": "example"
}
User login, if the email and password are correct the server will respond with a token
Delete /users/{_id}
Delete user,you need user token or admin token
